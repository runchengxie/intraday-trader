# 指标参考

本文解释回测和实盘绩效报告中的每个指标：计算公式、含义和解读基准。

## 基础指标

### 总收益率

计算公式：

```
总收益率 = (最终净值 - 初始资金) / 初始资金
```

这是最直观但信息量最少的指标。只看总收益率会忽略过程中的风险。

### 年化收益率

将总收益率按持有天数年化到 252 个交易日：

```
年化收益率 = (最终净值 / 初始资金) ^ (252 / 交易天数) - 1
```

年化后可以在不同时间跨度的回测之间做公平比较。

### 胜率

```
胜率 = 盈利交易次数 / 总交易次数
```

胜率高不等于策略好。一个胜率 90% 但每次赚 1%、亏 50% 的策略最终会亏钱。需要结合盈亏比一起看。

## 风险指标

### 最大回撤

从净值最高点到之后的最低点的最大跌幅：

```
最大回撤 = max(所有时刻 t 的 (峰值 - 当前净值) / 峰值)
```

它衡量的是最坏情况下你会亏多少。最大回撤 20% 意味着历史上某个时段你从最高点亏了 20%。

解读基准：

| 最大回撤 | 评价 |
| --- | --- |
| < 5% | 保守，接近低波动固收 |
| 5-15% | 中等风险，典型量化策略 |
| 15-30% | 高风险，需要有强 alpha 支撑 |
| > 30% | 极高风险，多数投资者难以承受 |

### VaR（风险价值）

在给定置信水平下，单日可能的最大亏损金额：

```
历史模拟法 VaR：
  VaR_95 = portfolio_value * percentile(日收益率序列, 5%)
```

例如 VaR=3% 意味着有 95% 的把握，单日亏损不会超过净值的 3%。

本项目的 RiskManager 支持两种计算方法：

- 历史模拟法：直接用历史收益率的分位数
- 参数法：假设收益率服从正态分布，用 z 值计算

参数法依赖正态假设，日内策略的收益率通常有肥尾，历史模拟法更可靠。

### CVaR（条件风险价值）

超过 VaR 阈值的那部分损失的平均值：

```
CVaR_95 = mean(收益率序列中 小于 VaR_95 的部分)
```

CVaR 回答"如果最坏情况发生了，平均会亏多少"。它比 VaR 更好地捕捉尾部风险。

## 风险调整收益

### 夏普比率

每承担一单位波动率获得的超额收益：

```
夏普比率 = (年化收益率 - 无风险利率) / 年化波动率
```

本项目无风险利率默认为 0（简化处理）。

解读基准：

| 夏普比率 | 评价 |
| --- | --- |
| < 0 | 跑输无风险利率，策略无效 |
| 0 - 0.5 | 勉强可接受 |
| 0.5 - 1.0 | 合理 |
| 1.0 - 2.0 | 良好 |
| > 2.0 | 优秀，但需要警惕过拟合或数据错误 |

注意事项：

- 夏普比率假设收益正态分布，日内策略的收益通常有肥尾和偏度，会高估夏普
- 回测夏普显著高于实盘夏普是正常的，实盘有额外的交易延迟、冲击成本和滑点
- 夏普对频率敏感：同样的策略在分钟级别可能显示 2.0+ 的夏普，日级别可能只有 0.5

## 交易成本与效率

### 换手率

衡量策略的交易活跃程度：

```
单边换手率 = 期间总成交金额 / 期间平均净值
年化换手率 = 单边换手率 * (252 / 期间交易天数)
```

解读：

- 低换手（< 50%）：低频策略，交易成本影响小
- 中换手（50-200%）：中等频率，需要关注佣金和滑点
- 高换手（> 200%）：高频策略，交易成本可能吃掉大部分 alpha

### 交易成本

```
交易成本 = 佣金合计 + 滑点合计 + 估计冲击成本
总成本率 = 交易成本 / 初始资金
```

如果总成本率接近甚至超过总收益率，说明策略的 gross alpha 被交易成本吃掉了。

## 集中度

### Herfindahl-Hirschman 指数（HHI）

```
HHI = sum(每个持仓的权重²)
```

- HHI 接近 1/n（n 是持仓数）：分散均匀
- HHI > 0.25：高度集中
- 单标的策略下 HHI 固定为 1.0

### 有效持仓数

```
有效持仓数 = 1 / HHI
```

例如 10 个等权重持仓的有效持仓数是 10。如果其中 1 个占 50%，其余平分，有效持仓数可能只有 3-4。

## 相对基准指标

### Alpha

策略收益中不能被基准收益解释的部分：

```
Alpha = 策略年化收益 - (无风险利率 + Beta * (基准年化收益 - 无风险利率))
```

Alpha 为正说明策略有超额收益。当前项目的 Alpha 计算简化了无风险利率（默认 0）。

### 信息比率

每承担一单位跟踪误差获得的超额收益：

```
信息比率 = 年化超额收益 / 跟踪误差（超额收益的标准差）
```

高信息比率说明策略不仅跑赢基准，而且跑赢得稳定（不是靠一两次大赌注）。

### Beta

策略收益对基准收益的敏感度：

```
Beta = Cov(策略收益, 基准收益) / Var(基准收益)
```

Beta=1 表示策略与基准同步波动，Beta=0.5 表示波动只有基准的一半。

## 实际回测输出示例

一次典型的 `intraday backtest run` 输出：

```
Buy & Hold Benchmark:
  Final Portfolio Value: $108,234.56
  Total Return: 8.23%
  Annual Return: 16.15%
  Sharpe Ratio: 0.85
  Max Drawdown: 12.4%
  VaR (95%): -1.85%
  CVaR (95%): -2.60%

Mean Reversion (Z-Score):
  Final Portfolio Value: $112,450.00
  Total Return: 12.45%
  Annual Return: 25.13%
  Sharpe Ratio: 1.32
  Max Drawdown: 7.8%
  Win Rate: 58.3%
  Profit Factor: 1.85
  Number of Trades: 47
  Turnover Rate: 280%
  Trading Costs: $890.45
  VaR (95%): -1.42%
  CVaR (95%): -2.01%
```

### 如何读懂这份输出

1. 均值回归策略跑赢了基准（12.45% vs 8.23%），年化后差距更大
2. 夏普 1.32 属于良好水平，但回测夏普通常高于实盘
3. 最大回撤 7.8% 比基准的 12.4% 更小，说明策略在下跌时的风控有效
4. 胜率 58.3% 不算特别高，但结合利润因子 1.85 看，赚钱时赚的比亏钱时多
5. 换手率 280% 意味着平均每 3-4 个月周转一次，交易成本 $890 相对净值 10 万是 0.89%，可以接受
6. VaR 和 CVaR 都低于基准，尾部风险控制更好

### 需要注意的陷阱

- 回测收益 ≠ 实盘收益。实盘有滑点、延迟、流动性限制和情绪影响
- 高夏普 + 低交易次数可能只是运气。47 笔交易在统计上不算多
- 换手率如果接近 1000%（年化），交易成本很可能高估了实际可成交的容量
