# 课程背景归档

本文合并了该项目在 CQF 课程阶段的三份原始文档，仅作历史参考。当前项目的功能、运行方式和开发规范以根目录 `README.md`、`AGENTS.md` 和 `docs/project-manual.md` 为准。

## 项目来源

本项目最初是 CQF（Certificate in Quantitative Finance）2025 年 1 月最终项目的提交内容，选题方向为"均值回归与趋势跟随算法交易"（课程提供的第 7 类选题）。

原始提交截止时间为 2025 年 8 月 21 日（英国夏令时），项目结果于 2025 年 11 月 13 日发布，通过分数线为 60%。

课程提供了 8 个选题方向：

1. 篮子产品信用利差
2. 深度学习资产预测
3. 配对交易策略设计与回测
4. Black-Litterman 与因子组合构建
5. 高级数据建模下的最优对冲
6. 分类模型的集成学习
7. 均值回归与趋势跟随算法交易（本项目）
8. 高维 PDE 的深度神经网络求解

## 课程要求摘录

以下内容整理自课程项目要求，已不再约束当前仓库的开发流程。

### 总体目标

围绕日内或较高频率数据，研究趋势跟随和均值回归策略：

- 用历史数据完成策略回测
- 解释指标、信号和参数选择
- 比较不同采样频率、窗口长度和市场状态下的表现
- 逐步扩展到接近实盘的 API 联调
- 处理订单状态、异常返回、部分成交和数据质量问题
- 讨论流动性、市场冲击和突发市场事件

### 代码使用原则

课程允许参考已有代码，但不允许直接提交课堂实验代码或只做轻微修改。可以复用成熟库（Backtrader、Pandas、NumPy 等），也可以从原理实现关键方法。核心模型、关键数值技术和研究结论必须能解释清楚。

### 报告结构建议

1. 引言：问题定义、研究设计、主要数据和数值技术表
2. 模型与方法：数学模型、数值方法、收敛性、精度和计算性质
3. 结果与分析：图表、指标、稳健性和敏感性分析
4. 结论：模型优缺点、实现限制和改进方向

报告不能只堆代码、表格和截图。每个图表都应解释它说明了什么，以及对结论有什么影响。

### 编程环境限制（仅限课程提交）

原课程对部分机器学习和深度学习题目要求使用 Python 和 TensorFlow/Keras API，并限制使用 PyTorch。这些约束是课程规则，与当前仓库无关。当前仓库的实际依赖以 `pyproject.toml` 为准。

### 学术诚信

课程要求所有提交内容必须由学生独立完成，并能解释清楚。直接复制课堂代码、共享脚本或他人内容，会被视为严重问题。原课程还限制使用 AI 写作或生成内容完成考试。

### 提交规则（仅限课程平台）

- 一份最终报告文件
- 一份包含代码、声明和辅助文件的压缩包
- 文件名应包含项目代码和注册姓名
- 不应提交零散脚本文件或泛泛命名的压缩包

## 与当前仓库的关系

本仓库已经演化为一个独立的 Python 项目。课程要求仅作为需求来源和设计背景保留，不约束当前的开发流程、测试规范或贡献方式。

代码的实际运行方式以 Makefile、Docker Compose 和 `intraday` CLI 命令为准。贡献方式以 PR、测试结果和代码评审为准。
